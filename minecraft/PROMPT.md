# 解説ページ生成プロンプト（Babylon Lite Minecraft デモ）

以下は `minecraft/index.html`（Babylon Lite の Minecraft デモ解説ページ）を
AI に一から作らせるためのプロンプトです。そのままコピペで使えます。

---

「Babylon Lite」公式リポジトリに含まれる Minecraft 風サンドボックスデモの
コード解説ページを、単一の HTML ファイル（index.html）として作ってください。
プログラミング初心者にもわかるやさしい日本語で、じっくり読ませる技術解説記事です。

## 全体方針
- 出力は完全に自己完結した1枚の HTML。CSS は <style> にインライン。
- 対象読者は「Minecraft みたいなゲームってどう作るの?」と思っている初学者。
  専門用語は必ず噛み砕き、たとえ話を添える。
- 実在のデモ解説という体裁。数字や仕様は下記「事実」に忠実に。

## デザイン
- テイスト: Minecraft を思わせるドット絵フォント × 落ち着いた自然色。
  紙のような明るい背景 (#F3F6EF)、白いカード、細い緑がかった罫線。
- フォント: 見出しは DotGothic16、本文は Zen Kaku Gothic New、
  コードは M PLUS 1 Code。Google Fonts から読み込む。
- カラーパレットを CSS 変数で定義: 草(緑)/土(茶)/石(灰)/水(青)/
  グロウ(橙)/豚(赤茶) の6系統、それぞれ通常色・濃色・淡い背景色をもつ。
- ヒーロー: 方眼紙のような薄いグリッド背景 + 空色グラデーション。
  「BABYLON LITE 公式デモを読む」の小見出し、大きなタイトル、リード文、
  4つの統計 (25 ファイル / 4,669 行 / 同梱アート素材 0枚 / 公開APIのみ 100%)。
  右側に草・石・水・グロウの アイソメトリック立方体をインライン SVG で4個描き、
  上下にゆっくり揺れるアニメ (bob)。
- 本文は2カラム: 左に position:sticky の目次(もくじ)、右に本文。
  目次の各項目には章の色を表す小さな四角チップを付ける。
- カード/用語ボックス/コールアウト(3色: 標準=橙, water=青, grass=緑)/
  コード表示(濃い緑地に薄緑文字)/表/kbd キー表記 のスタイルを用意。
- レスポンシブ: 920px 以下で1カラム化、目次は横スクロールのバーに。
  prefers-reduced-motion でアニメ停止。

## 図版
- 処理の流れは mermaid.js (CDN, ESM 動的 import) の flowchart で描く。
  テーマは base をカスタムし、上のパレットに合わせた themeVariables を指定。
- 複雑な図(チャンクの俯瞰、水の世代拡散、箱で組んだ牛)は
  手描き風のインライン SVG で描き、figure/figcaption で囲む。

## 章構成（全15章）
1. このデモとは — 3つの特徴(種から生成/アート素材ゼロ/全処理に予算)
2. あそびかた — 操作表 (WASD, Space, マウス, 左右クリック, 1〜0/Tab,
   Ctrl+S/O, F3) と かくし機能 (?time=0.75 等の URL パラメータ)
3. 用語ミニ辞典 — ボクセル/チャンク/メッシュ/面カリング/AO/fBmノイズ/
   BFS/シェーダ・WGSL/テクスチャアトラス/決定論 を2列カードで
4. 全体の地図 — minecraft.ts(236行) を指揮者に見立て、
   機能層/描画層/ワールド生成・照明層/基盤層 の4層依存図 (mermaid)
5. 起動の流れ — エンジン→アトラス生成→部品生成→warmAround→ループ開始 の
   5ステップ図 + 13×13 チャンク先読みの俯瞰 SVG(霧の半径も図示)
6. 地形のつくりかた — seed(1337) から高さ・気温・湿度ノイズ→バイオーム→
   洞窟・鉱石・木 を生成する流れ
7. 光のしくみ — スカイライト + ブロックライトの2チャネル、BFSで拡散
8. ブロックが絵になるまで — mesher が見える面だけ三角形化、頂点AO
9. 毎フレームの仕事 — 入力→物理→processQueue→lighting→mobs等→HUD の
   ループ図 と「予算」一覧表(メッシュ3/光4/水4step等) + dtクランプの注意
10. 壊す・置くと何が起きる — 左クリックで raycast→setBlock→光無効化→
    remesh→water→falling→粒エフェクト が連鎖する図。境目だけ隣を再生成
11. 水と砂の物理 — 水は0.2秒ごとに1リングずつ広がる「世代」方式(拡散SVG図)、
    砂・砂利はエンティティ化して落下(すり抜け/重なり/柱くずれ対策)
12. 動物たち — mobs.ts(491行)。牛豚羊にわとり、画像素材ゼロで箱を組み立て、
    股関節ピボットで足を振る。牛を箱で分解した SVG 図 + 仕様表(最大10体等)
13. セーブ&ロード — 保存するのは seed+time+player+編集差分だけ。
    SaveData interface のコード例。File System Access API
14. 3つの設計テーマ — 決定論 / アセットゼロ / 予算駆動。
    コメントが「歴史書」になっている点も
15. ファイル一覧 — 全25ファイルを層ごとに区切った表(ファイル名/行数/役割)

## 事実（正確に反映すること）
- WebGPU 専用の軽量エンジン、全25ファイル・約4,669行
- チャンク 16×16×96、海面 y=30、seed=1337、霧 48ブロックから
- ブロック23種、動物 最大10体・10〜44ブロック先に出現・96で消滅
- 予算: メッシュ3チャンク/フレーム、光4回、水4ステップ(1step 8192セル)、
  落下1024セル。dt は最大0.1秒にクランプ

## 使い方のコツ
- 情報量が多く途中で息切れする場合は「CSS+ヒーロー+目次」「第1〜7章」
  「第8〜15章+フッター」のように分割依頼すると安定する。
- 図を軽くしたい場合は「複雑な SVG 図は省略し mermaid と表だけで」と足す。
